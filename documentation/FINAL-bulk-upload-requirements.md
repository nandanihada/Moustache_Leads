# Bulk Upload Requirements - FINAL

## Critical Understanding

There are TWO different identifiers in the system:
1. **offer_id** - Your internal system's unique offer identifier (auto-generated by the system)
2. **campaign_id** - The network's campaign/offer identifier (from the external network, provided in spreadsheet)

## Always Required Fields (ALL Networks)

These fields are **MANDATORY** for every offer, regardless of network:

### 1. campaign_id (also called offer_id in spreadsheet)
- **What it is**: The network's campaign/offer identifier
- **Required for**: ALL platforms (100% mandatory)
- **Example**: `12345`
- **Column names accepted**: `campaign_id`, `offer_id`

### 2. name (also called title)
- **What it is**: The offer name/title
- **Required for**: ALL platforms
- **Example**: `Sign up for Premium Subscription`
- **Column names accepted**: `name`, `title`

### 3. description
- **What it is**: Offer description
- **Required for**: ALL platforms
- **Example**: `Complete a survey and earn rewards`
- **Column names accepted**: `description`

### 4. network (also called platform)
- **What it is**: The network/platform name
- **Required for**: ALL platforms
- **Example**: `cpamerchant`, `leadads`, `chameleonads`, `othernetwork`
- **Column names accepted**: `network`, `platform`, `name of platform`

### 5. countries
- **What it is**: Target countries (comma-separated)
- **Required for**: ALL platforms
- **Example**: `US, UK, CA`
- **Column names accepted**: `countries`, `country`

### 6. payout OR revenue_share_percent
- **What it is**: Either fixed payout amount OR revenue share percentage
- **Required for**: ALL platforms (one of them)
- **Examples**: `$5.00`, `50%`, `€10`
- **Column names accepted**: `payout`, `revenue_share_percent`, `percent`

## Conditionally Required: target_url

The `target_url` field requirement depends ONLY on the network type:

### Special Networks (target_url OPTIONAL)
For these three networks, `target_url` is **OPTIONAL**:
- **cpamerchant**
- **leadads**
- **chameleonads**

**Why?** The system automatically generates the tracking URL using the campaign_id.

**Auto-generated URLs:**
- cpamerchant: `https://tracking.cpamerchant.com/aff_c?offer_id={campaign_id}&aff_id=3394`
- leadads: `https://leadads.go2jump.org/aff_c?offer_id={campaign_id}&aff_id=10843`
- chameleonads: `https://chameleonads.go2cloud.org/aff_c?offer_id={campaign_id}&aff_id=5696`

**Example (cpamerchant without target_url):**
```csv
campaign_id,name,description,network,countries,payout
12345,Survey Offer,Complete a survey,cpamerchant,US,5.00
```
✅ Valid - target_url will be auto-generated

**Example (cpamerchant with custom target_url):**
```csv
campaign_id,name,target_url,description,network,countries,payout
12345,Survey Offer,https://custom.url/offer,Complete a survey,cpamerchant,US,5.00
```
✅ Valid - custom target_url will be used instead of auto-generated

### All Other Networks (target_url REQUIRED)
For any network that is NOT one of the three special networks, `target_url` is **REQUIRED**.

**Example (regular network):**
```csv
campaign_id,name,target_url,description,network,countries,payout
12345,Test Offer,https://example.com/offer,Complete survey,othernetwork,US,5.00
```
✅ Valid - all required fields present

**Example (regular network - INVALID):**
```csv
campaign_id,name,description,network,countries,payout
12345,Test Offer,Complete survey,othernetwork,US,5.00
```
❌ Invalid - target_url is missing for non-special network

## Common Mistakes

### ❌ Mistake 1: Thinking campaign_id is optional for regular networks
```csv
name,target_url,description,network,countries,payout
Test Offer,https://example.com,Test description,regularnetwork,US,5.00
```
**Error**: Missing campaign_id
**Fix**: campaign_id is REQUIRED for ALL platforms

### ❌ Mistake 2: Missing target_url for regular networks
```csv
campaign_id,name,description,network,countries,payout
12345,Test Offer,Test description,regularnetwork,US,5.00
```
**Error**: Missing target_url
**Fix**: target_url is REQUIRED for non-special networks

### ✅ Correct: Special network without target_url
```csv
campaign_id,name,description,network,countries,payout
12345,Test Offer,Test description,cpamerchant,US,5.00
```
**Valid**: target_url will be auto-generated for cpamerchant

### ✅ Correct: Regular network with all fields
```csv
campaign_id,name,target_url,description,network,countries,payout
12345,Test Offer,https://example.com/offer,Test description,regularnetwork,US,5.00
```
**Valid**: All required fields present

## Validation Logic Flow

```
1. Check if ALL base required fields are present:
   - campaign_id ✓
   - name ✓
   - description ✓
   - network ✓
   - countries ✓
   - payout OR revenue_share_percent ✓

2. Check network type:
   IF network is cpamerchant, leadads, or chameleonads:
      - target_url is OPTIONAL
      - If missing, will be auto-generated from campaign_id
      - Warning: "target_url will be auto-generated"
   ELSE:
      - target_url is REQUIRED
      - If missing, validation fails

3. Result:
   - Valid: All required fields present
   - Invalid: Missing one or more required fields
```

## Key Takeaways

1. **campaign_id is ALWAYS required** - for every single platform
2. **target_url is conditionally required** - depends on network type
3. **Special networks** (cpamerchant, leadads, chameleonads) can auto-generate target_url
4. **Regular networks** must have target_url provided
5. **All other fields** (name, description, network, countries, payout) are always required
